# PianoBot — инструкция по использованию

Этот бот в Telegram помогает сотрудникам учитывать рабочее время по задачам (модель/деталь/процесс), а администраторам — управлять сотрудниками, справочниками и отчётами.

## 1) Быстрый старт для всех

1. Откройте бота в Telegram.
2. Нажмите **/start**.
3. Если вы не авторизованы, бот покажет ваш Telegram ID и попросит передать его администратору.
4. После добавления в систему нажмите **/start** снова.

> Полезно: команда **/id** показывает ваш Telegram ID.

---

## 2) Инструкция для обычного пользователя (сотрудника)

После входа у сотрудника открывается меню:
- **Start a new task** — начать новую задачу.
- **Continue the task** — продолжить ранее начатую задачу.

### 2.1 Начать новую задачу

1. Нажмите **Start a new task**.
2. Выберите:
   - модель,
   - деталь,
   - процесс.
3. Нажмите на нужный вариант, после чего задача запускается.
4. Бот покажет карточку задачи и кнопки:
   - **Pause** — поставить на паузу (время сохранится),
   - **Finish** — завершить и записать отчёт.

### 2.2 Если нужной задачи нет в списке

1. В выборе модели нажмите **Other**.
2. Введите название задачи.
3. Введите комментарий/описание.
4. Подтвердите кнопкой **ok**.

### 2.3 Продолжить задачу

1. Нажмите **Continue the task**.
2. Выберите задачу из списка начатых.
3. После выбора снова доступны **Pause** и **Finish**.

### 2.4 Сортировка и навигация

- В списках доступны:
  - **Sort by date** / **Sort by name**,
  - переключение страниц **⬅️ / ➡️**,
  - кнопка **Back** для возврата.

### 2.5 Что важно помнить

- Пока задача активна, время продолжает идти.
- Если хотите временно остановиться — нажимайте **Pause**.
- Чтобы время попало в отчёты, задачу нужно завершить через **Finish**.

## 3) Инструкция для администратора

После входа администратору доступно меню:
- **Who works now**
- **Reports**
- **HR management**
- **Backup .DB**
- **User mode**

Также администратор может переключиться в режим сотрудника и обратно (если его аккаунт имеет оба права).

### 3.1 Who works now

Показывает, кто сейчас работает и над какой задачей (модель/деталь/процесс).

### 3.2 HR management (управление персоналом)

Раздел содержит:
- **List of employees** — список сотрудников;
- **Add user by ID** — добавить сотрудника по Telegram ID.

#### Добавление сотрудника
1. Нажмите **Add user by ID**.
2. Введите Telegram ID (только цифры).
3. Введите имя пользователя.

#### Редактирование сотрудника из списка
В карточке сотрудника можно:
- изменить роль (**Make admin** / **Make user**),
- удалить из системы (**Fire**).

### 3.3 Reports (работа с отчётами)

В разделе **Reports** доступны:
- просмотр последних записей,
- поиск отчёта по ID (**Find by ID**),
- редактирование полей отчёта,
- переход в меню скачивания (**Download reports**),
- переход в **Settings**.

#### Редактирование отчёта
По ID отчёта можно изменить:
- имя,
- модель,
- деталь,
- процесс,
- время (в минутах),
- дату.

Формат даты при вводе: **dd.mm.yyyy**.

### 3.4 Download reports (выгрузка отчётов)

Администратор может выгрузить:
- **For period download reports** — за период,
- **For user download reports** — по сотруднику,
- **For user and period download reports** — по сотруднику за период,
- **Download all reports** — все отчёты,
- **Comparison table** — сравнительная таблица по моделям.

Формат периода: **dd.mm.yyyy - dd.mm.yyyy**
(пример: `19.09.2025 - 19.10.2025`).

Файлы приходят в формате Excel (`.xlsx`).

### 3.5 Reports → Settings

В этом разделе администратор управляет справочниками:
- **List of pianos** (модели),
- **List of details** (детали),
- **List of processes** (процессы).

Для каждой записи доступны действия:
- **Add**,
- **Edit**,
- **Delete**.

Также справочники можно массово выгружать и загружать Excel-файлом:
- **Export catalog xlsx** — скачать текущие модели, детали и процессы;
- **Import catalog xlsx** — загрузить `.xlsx` с обновлениями справочников.

Файл справочников состоит из трёх листов:

#### Лист `Models`
| id | name |
|---:|------|
| 1 | Model A |
|  | Model C |

#### Лист `Parts`
| id | name |
|---:|------|
| 1 | Keyboard |
|  | Case |

#### Лист `Processes`
| id | part_id | part_name | name |
|---:|--------:|-----------|------|
| 1 | 1 | Keyboard | Sanding |
|  | 2 | Pedal | Assembly |

Правила импорта:
- пустой `id` добавляет новую запись;
- заполненный существующий `id` обновляет запись;
- процессы должны быть связаны с деталью через `part_id` или уникальный `part_name`;
- если в файле есть ошибки, изменения не применяются;
- перед успешным импортом бот создаёт резервную копию базы.

---

## 4) Восстановление прав главного администратора

Есть специальная служебная команда:
- отправьте боту текст **reset administrator**
- команда работает только для ID главного администратора, заданного в переменных окружения.

После выполнения права администратора восстанавливаются.

---

## 5) Краткий сценарий работы

### Сотрудник
`/start` → `Start a new task` → выбор модели/детали/процесса → `Pause`/`Finish`.

### Администратор
`/start` → `HR management` (добавить пользователей) → `Reports` (контроль и выгрузка) → `Settings` (актуализация справочников).

---

## 6) Частые проблемы

- **«Вы не авторизованы»** — передайте свой ID администратору и попросите добавить вас.
- **Не находится отчёт по ID** — проверьте, что введено целое число без пробелов и символов.
- **Ошибка периода** — используйте формат строго `dd.mm.yyyy - dd.mm.yyyy`.
- **Пустая выгрузка** — по выбранным фильтрам нет данных.
